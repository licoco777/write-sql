# 附件号码清单查实收按月打横

来源：SC-009 路径1 验证实例

> 可复用专项规则见 `scenarios/SC-009_047117收入实收查询.md`；本文件为 **路径1：附件清单 → 117** 的已验证编排。

## 适用

- 用户提供 **号码/接入号附件清单**（含序号），查指定账期区间的 **实收金额**。
- 输出需要 **按月打横**（如 `m202501`、`m202502`）或与附件行一一对齐。

## 不适用

- 无附件，需按营服/产品/双线等条件 **先圈号再查实收** → `VC-20260611-002`（路径2）。
- 查客户基本面/产数 → `VC-20260611-003` 或 047 路径。
- 查科目级税后收入 → 048 / `VC-20260520-001`。

## 主表与口径

- **驱动表**：用户附件种子表（保留 `seq_no`、`acc_nbr`）。
- **收入事实表**：117 `zone_gz_yz.dwd_yz_if_real_src_sum_new_final`。
- **账期**：`par_month_id BETWEEN ${start_month} AND ${end_month}`。
- **实收公式**：

```text
sum(case when flag = 'HF' then amount - amount_tc else 0 end)
+ sum(case when flag = 'OT' then amount else 0 end) as ss_amount
```

## 字段映射

| 需求字段 | 来源 |
|----------|------|
| 序号 | 附件 `seq_no` |
| 号码 | 附件 `acc_nbr` |
| 各月实收 | 117 按公式汇总后 `SUM(CASE WHEN par_month_id=YYYYMM THEN ss_amount ELSE 0 END)` |

## 补表

- 默认不补表；附件 LEFT JOIN 117 保行。
- 若需带出营服/客户名，可 LEFT JOIN 069（同账期 `acc_nbr`），不改变实收度量来源。

## SQL 编排

两步 CTAS：Step1 先汇总 117，Step2 回连附件并按月打横。

```sql
drop table if exists tmp_acc_nbr_real_pay_month purge;
create table tmp_acc_nbr_real_pay_month
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
    acc_nbr,
    par_month_id,
    sum(case when flag = 'HF' then amount - amount_tc else 0 end)
        + sum(case when flag = 'OT' then amount else 0 end) as ss_amount
from zone_gz_yz.dwd_yz_if_real_src_sum_new_final
where par_month_id between ${start_month} and ${end_month}
group by acc_nbr, par_month_id;

drop table if exists ads_acc_nbr_real_pay_pivot purge;
create table ads_acc_nbr_real_pay_pivot
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
  a.seq_no,
  a.acc_nbr,
  sum(case when s.par_month_id = ${m202501} then s.ss_amount else 0 end) as m202501,
  sum(case when s.par_month_id = ${m202502} then s.ss_amount else 0 end) as m202502
from tmp_acc_nbr_list a
left join tmp_acc_nbr_real_pay_month s
    on a.acc_nbr = s.acc_nbr
group by a.seq_no, a.acc_nbr
order by a.seq_no;

select count(*) as result_rows, count(distinct acc_nbr) as result_acc_cnt
from ads_acc_nbr_real_pay_pivot;
```

## 自检

- 附件输入行数 = 最终输出行数（LEFT JOIN 保行）。
- `select count(distinct acc_nbr) from ss` 与 117 命中数合理。
- 样例 `limit 10` 检查打横列非异常 NULL。
