# 按年科目清单匹配科目级税后收入

来源：write-query 交互验证（政企科目维表 + 三年分列匹配）

## 适用

- 用户按 **外部科目维表**（`due_income_code23/24/25` 分列）匹配 **SR 科目税后收入**。
- 账期按 **年** 汇总（`substr(par_month_id,1,4)`），输出局向、营服、产权客户、产品、揽装人等维度 + `sum(fee_all)`。
- 用户采用 **按年建临时表 + 按年 JOIN 维表列** 的生产写法。

## 不适用

- 服务级划小收入汇总（无科目维度）→ 047 最终版划小收入、`sum(a0)`。
- 入网、到达、积分、续约类需求。

## 主表与口径

- **主表**：048 全量科目级收入 `dwm_srhx_src_income_list_mon`（或分局视图 `zone_gz.view_{分局}_ads_srhx_src_income_list_mon`）。
- **粒度**：号码/服务级明细（每行一个 `fee_all`）；汇总时对输出维度 `sum(fee_all)`。
- **度量**：`sum(fee_all)`（税后收入）；用户口头「确认收入（税后）」在本场景对齐 `fee_all`。
- **分区**：`par_month_id`（月，如 `202301`）；账期年由 `substr(cast(par_month_id as string), 1, 4)` 派生。
- **常用过滤**：`contract_flag = 1`（划小）、`is_filter = 0`（非主营科目，参考 `M-TOPIC-REV-006`）。
- **营服**：默认 **划小营服** `branch_name`，不用 `channel_branch_name`（揽装营服）。

## 字段映射

| 需求字段 | 来源 |
|----------|------|
| 账期（年） | `substr(cast(par_month_id as string), 1, 4)` |
| 划小局向 | `subst_name` |
| 所属营服 | `branch_name` |
| 产权客户名称 | `cust_name` |
| 产权客户编码 | `cust_nbr` |
| 产品名称 | `prod_name` |
| 揽装人 | `sales_name` |
| 揽装人编码 | `sales_code` |
| 确认收入（税后） | `sum(fee_all)` |

## 补表

- **科目维表**（用户自建）：`ads_yz_zqb_xc_due_income_code_dim_list`
  - `due_income_code23` / `due_income_code24` / `due_income_code25` 分别对应 2023/2024/2025 年 SR 科目白名单。
  - 按年 JOIN：23 年 `a.due_income_code = b.due_income_code23` 且 `par_month_id` 当年 1–12 月；24/25 年同理。
  - 维表多行同码会放大明细行，汇总前宜按年列去重或接受按年拆临时表降低风险。

## SQL 编排

### 推荐模式：按年临时表 + UNION 汇总

1. 建维表（从配置表导入三列科目编码）。
2. `tmp_..._23`：`048` INNER JOIN 维表 `due_income_code23`，`par_month_id` 202301–202312，加 `contract_flag`/`is_filter`。
3. `tmp_..._24`：JOIN `due_income_code24`，`par_month_id` 202401–202412。
4. `tmp_..._25`：JOIN `due_income_code25`，`par_month_id` 202501–202512。
5. 汇总：`union all` 三段后 `group by` 账期年 + 维度 + `sum(fee_all)`；或在各年子查询内先 `sum(fee_all)` 再 `union`（与外层 sum 等价）。

### 汇总注意

- 临时表为 **号码级** 时，内层 `select ... fee_all` 可不 sum，但 **交付必须** 有 `group by 维度 + sum(fee_all)`（内层或外层至少一层）。
- 内外都不 sum 会同一维度多行，不是最终汇总结果。

## 参考 SQL 骨架

```sql
-- 23年明细
create table tmp_ads_yz_zqb_xc_code_cust_sr_dwb_23 as
select a.*
from dwm_srhx_src_income_list_mon a
join ads_yz_zqb_xc_due_income_code_dim_list b
  on a.due_income_code = b.due_income_code23
where a.par_month_id >= 202301 and a.par_month_id <= 202312
  and a.contract_flag = 1 and a.is_filter = 0;

-- 汇总（外层 sum 示例）
select acct_year, subst_name, branch_name, cust_name, cust_nbr,
       prod_name, sales_name, sales_code, sum(fee_all) as 确认收入_税后
from (
  select '2023' as acct_year, subst_name, branch_name, cust_name, cust_nbr,
         prod_name, sales_name, sales_code, fee_all
  from tmp_ads_yz_zqb_xc_code_cust_sr_dwb_23
  union all
  -- 24、25 年同理
) t
group by acct_year, subst_name, branch_name, cust_name, cust_nbr,
         prod_name, sales_name, sales_code;
```
