# 圈定双线号码后查实收（CTAS）

来源：SC-009 路径2 验证实例

> 可复用专项规则见 `scenarios/SC-009_047117收入实收查询.md`；本文件为 **路径2：无附件、069 圈定 → 117** 的已验证编排。

## 适用

- **无附件**；用户给出业务条件（如某营服、互联网/组网专线、在网状态），先 **圈定号码** 再查 **实收金额**。
- 圈定条件依赖 **069**（如 `prod_type2 IN (60,70,71)`、`branch_id`、`is_cancel_user=0`）。
- 对象范围较大或需保留中间结果供用户改条件 → **CTAS 两步**。

## 不适用

- 用户已提供号码附件清单 → `VC-20260611-001`（路径1）。
- 仅要「某营服某月实收汇总」且 117 上可直接 `WHERE branch_id` → SC-009 路径3（可单 SELECT）。
- 圈定后要科目明细 → 048，非 117。

## 主表与口径

- **Step1 圈定**：069 `dwm_yz_tb_comm_cm_all_final`（或月表，按账期窗口选择）。
- **Step2 收入**：117 `zone_gz_yz.dwd_yz_if_real_src_sum_new_final`。
- **双线定义**：069 `prod_type2 in (60, 70, 71)`（60=互联网专线，70/71=组网专线）。
- **账期**：圈定快照 `par_month_id = ${snap_month}`；117 取数 `par_month_id between ${start_month} and ${end_month}`。

## 字段映射

| 需求字段 | 来源 |
|----------|------|
| 号码 | Step1 `acc_nbr` |
| 服务标识 | Step1 `serv_id` |
| 营服 | Step1 `branch_id`、`branch_name` |
| 实收（按月） | Step2 117 公式汇总后打横 |

## 补表

- 圈定阶段不补表；双线速率/月租若需要另按 FIELD_BACKFILL 补 033，与本案例实收无关。

## SQL 编排

```sql
-- Step1：069 圈定双线号码（示例：某营服 + 在网快照月）
drop table if exists tmp_scope_sx_acc purge;
create table tmp_scope_sx_acc
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select distinct
  acc_nbr,
  serv_id,
  branch_id,
  branch_name
from dwm_yz_tb_comm_cm_all_final
where par_month_id = ${snap_month}
  and branch_id = ${branch_id}
  and prod_type2 in (60, 70, 71)
  and is_cancel_user = 0;

select count(*) from tmp_scope_sx_acc;

-- Step2：圈定结果 JOIN 117，先号码+月汇总再打横
drop table if exists tmp_scope_sx_ss purge;
create table tmp_scope_sx_ss
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
  s.acc_nbr,
  s.serv_id,
  s.branch_id,
  s.branch_name,
  r.par_month_id,
  sum(case when r.flag = 'HF' then r.amount - r.amount_tc else 0 end)
    + sum(case when r.flag = 'OT' then r.amount else 0 end) as ss_amount
from tmp_scope_sx_acc s
inner join zone_gz_yz.dwd_yz_if_real_src_sum_new_final r
  on s.acc_nbr = r.acc_nbr
 and r.par_month_id between ${start_month} and ${end_month}
group by s.acc_nbr, s.serv_id, s.branch_id, s.branch_name, r.par_month_id;

select count(*) from tmp_scope_sx_ss;
```

外层可按 `branch_id` 汇总或按号码明细输出；月份打横用 `SUM(CASE WHEN par_month_id=YYYYMM THEN ss_amount ELSE 0 END)`。

## 自检

- Step1 `count(*)` 与业务预期双线规模量级一致。
- Step2 行数 ≤ Step1 号码数 × 月数（内层已按 `acc_nbr, par_month_id` 汇总时应等于命中号码×有实收月数）。
- 确认未把附件文件名中的「VPN/专线」当作 SQL 条件；双线范围必须在 Step1 用 `prod_type2` 显式写出。
