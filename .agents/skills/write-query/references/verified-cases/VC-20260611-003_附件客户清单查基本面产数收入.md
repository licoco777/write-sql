# 附件客户清单查基本面/产数收入

来源：SC-009 路径1（047）验证实例

> 可复用专项规则见 `scenarios/SC-009_047117收入实收查询.md`；本文件为 **路径1：附件 cust_nbr → 047（先客户级聚合）** 的已验证编排。

## 适用

- 用户提供 **客户编码 `cust_nbr` 附件清单**（含序号），查指定账期的 **基本面收入** `fee_fm_new` 和/或 **产数收入** `fee_cs`。
- 需要按月打横或与附件客户行对齐。

## 不适用

- 查 **实收金额** → 117 / `VC-20260611-001`。
- **标准指标**「基本面收入怎么算」（按营服/包区报表）→ `METRIC_INDEX` + `VC-20260502-005`。
- 仅客户名无编码的兜底匹配 → 须提示重名风险，优先让用户补 `cust_nbr`。

## 主表与口径

- **驱动表**：用户附件种子表（`seq_no`、`cust_nbr`）。
- **收入事实表**：047 `dwm_srhx_serv_list_mon_final`。
- **账期**：`par_month_id BETWEEN ${start_month} AND ${end_month}`。
- **聚合铁律**：047 为服务/月粒度；**必须先按 `cust_nbr, par_month_id` 聚合**，再回连附件。

## 字段映射

| 需求字段 | 来源 |
|----------|------|
| 序号 | 附件 `seq_no` |
| 客户编码 | 附件 `cust_nbr` |
| 基本面收入 | `sum(fee_fm_new)`（客户级） |
| 产数收入 | `sum(fee_cs)`（客户级） |
| 月份打横 | `SUM(CASE WHEN par_month_id=YYYYMM THEN fee_fm_new ELSE 0 END)` |

## 补表

- 不直接把 047 服务级明细 JOIN 附件；必须经客户级聚合 CTE/子查询。

## SQL 编排

```sql
drop table if exists tmp_cust_nbr_income_month purge;
create table tmp_cust_nbr_income_month
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
    cust_nbr,
    par_month_id,
    sum(fee_fm_new) as fee_fm_new,
    sum(fee_cs) as fee_cs
from dwm_srhx_serv_list_mon_final
where par_month_id between ${start_month} and ${end_month}
group by cust_nbr, par_month_id;

drop table if exists ads_cust_nbr_income_pivot purge;
create table ads_cust_nbr_income_pivot
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
  a.seq_no,
  a.cust_nbr,
  sum(case when c.par_month_id = ${m202501} then c.fee_fm_new else 0 end) as fm_m202501,
  sum(case when c.par_month_id = ${m202501} then c.fee_cs else 0 end) as cs_m202501,
  sum(case when c.par_month_id = ${m202502} then c.fee_fm_new else 0 end) as fm_m202502,
  sum(case when c.par_month_id = ${m202502} then c.fee_cs else 0 end) as cs_m202502
from tmp_cust_nbr_list a
left join tmp_cust_nbr_income_month c
    on a.cust_nbr = c.cust_nbr
group by a.seq_no, a.cust_nbr
order by a.seq_no;

select count(*) as result_rows, count(distinct cust_nbr) as result_cust_cnt
from ads_cust_nbr_income_pivot;
```

## 自检

- 内层 `cust_inc`：`count(distinct cust_nbr)` 与 047 命中客户数一致。
- 附件行数 = 输出行数（LEFT JOIN）。
- 对比未聚合直接 JOIN 047 的行数，确认无服务级膨胀。
