# 047/117 收入实收取数

## 适用

- 查 **实收金额**（117 实收来源汇总表）。
- 查 **客户基本面收入** `fee_fm_new` 或 **产数收入** `fee_cs`（047 最终版划小收入）。
- 对象范围可以是：**用户附件清单**、**业务条件圈定（无附件）**、**按维度直接在收入表汇总**、**种子 serv_id/临时表驱动**。
- 典型触发词：实收、实收金额、实收来源、HF/OT、基本面收入（客户清单语境）、产数收入、客户清单查收入、按号码/接入号/客户编码查收入、先圈号码再查实收、VPN/专线查实收（须显式圈定条件）。

## 不适用

- **标准指标报表口径**（如「基本面收入这个指标怎么算」「划小收入指标」）→ `METRIC_INDEX.md` + 单指标文件；收入多维模板见 `verified-cases/VC-20260502-005`。
- **SR 科目 / 账目项 / 税后科目明细**（`due_income_name`、`fee_all` 等）→ **048** 全量科目级收入；圈定后查科目见 `ROUTING.md`「项目/客户号码圈定后查账目项收入明细」、`verified-cases/VC-20260520-001`。
- **台阶收入** → 101；**097 基本面月清单专项口径** → 097 + 指标文件。
- 用 **069 `fee` / `fee_new_tax`** 或 **047/048 税后收入** 替代 **117 实收**。
- 用 **117/047** 替代 **048 科目明细**。

## 收入表选择

| 用户要什么 | 事实表 | 生产表名 | 账期字段 | 度量 |
|---|---|---|---|---|
| 实收金额 / 实收来源 | 117 实收来源汇总表 | `zone_gz_yz.dwd_yz_if_real_src_sum_new_final` | `par_month_id` | 见下文「117 实收公式」 |
| 客户基本面 / 产数收入（客户清单语境） | 047 最终版划小收入 | `dwm_srhx_serv_list_mon_final` | `par_month_id` | `sum(fee_fm_new)` / `sum(fee_cs)` |
| 科目级税后收入 | 048 | `dwm_srhx_src_income_list_mon` | `month_id` | `sum(fee_all)` → **不在本 SC** |

## 入口路径判定

先判断 **收入类型（117 vs 047）**，再判断 **对象范围怎么来**：

```text
收入类型？
  实收 → 117
  客户基本面/产数 → 047
  科目明细 → 048（退出本 SC）

对象范围？
  路径1：用户附件清单（acc_nbr / cust_nbr / serv_id）
  路径2：无附件，业务条件圈定（069 / 033 / 022 等先筛对象）
  路径3：无清单，按维度直接在收入表汇总（117/047 字段足够时）
  路径4：种子 serv_id / 临时表驱动（全程 LEFT JOIN 保种子行）
```

| 路径 | 何时使用 | 编排建议 |
|---|---|---|
| **路径1 附件清单** | 用户提供号码/客户编码清单，或要求与附件行对齐 | 保留附件序号；核对输入/命中/输出行数 |
| **路径2 先圈后查** | 无附件；按营服、产品、客户名、双线、VPN/专线等条件筛对象再查收入 | **默认 CTAS 两步**：Step1 圈 `acc_nbr`/`serv_id`/`cust_nbr` → Step2 JOIN 117/047 |
| **路径3 直查收入表** | 无预定义清单；过滤维度在 117/047 上可直接表达（如营服、产品已有字段） | 单 SELECT 或简单 CTAS；`WHERE par_month_id BETWEEN ...` + `GROUP BY` |
| **路径4 种子驱动** | 用户给 serv_id 种子表，查对应实收/收入 | 种子 LEFT JOIN；复杂时 CTAS |

**路径2 vs 路径3 判定**：若产品分类、客群、双线定义、VPN 范围等条件**只在 069/033** 上能准确表达 → 走路径2；若用户仅要「某营服某月实收汇总」且 117/047 已有 `branch_id` 等字段 → 可路径3。不确定时先方案确认。

## 117 实收规则

### 实收公式（唯一权威）

```sql
sum(case when flag = 'HF' then amount - amount_tc else 0 end)
+ sum(case when flag = 'OT' then amount else 0 end) as ss_amount
```

### JOIN 与聚合

- 账期：`par_month_id BETWEEN ${start_month} AND ${end_month}`（闭区间，禁止只写 `>=`）。
- 匹配键：优先 `acc_nbr`；有 `serv_id` 时可用 `serv_id + par_month_id` 提高准确性。
- **先汇总再打横**：内层按 `par_month_id, serv_id, acc_nbr`（或输出维度）`sum(ss_amount)`，外层再 `SUM(CASE WHEN par_month_id=YYYYMM THEN ss_amount ELSE 0 END)` 打横。
- 附件标题中的「VPN/专线」**不是**已校验的产品属性；若用户要求校验产品范围，Step1 用 069（`prod_type2 IN (60,70,71)` 等）或 033 显式圈定后再 JOIN 117。

### 路径示例（无附件圈定后查实收）

```text
Step1 tmp_scope：069 WHERE 营服/产品/客户/双线条件 → acc_nbr, serv_id
Step2：tmp_scope JOIN 117 ON acc_nbr（+ par_month_id）→ 按公式 sum → 按输出维度/月份打横
```

## 047 客户收入规则

### 度量

- 基本面收入：`sum(fee_fm_new)`
- 产数收入：`sum(fee_cs)`

### 聚合铁律

- 047 是 **服务/月粒度**；对客户清单或 `cust_nbr` 维度输出时，**必须先按 `cust_nbr`（+ 需要的 `par_month_id`）聚合到客户级**，再回连附件或分组打横。
- 禁止把 047 服务/月明细直接 JOIN 附件 `cust_nbr` 导致行数膨胀。

### JOIN 与打横

- 账期：`par_month_id BETWEEN ${start_month} AND ${end_month}`。
- 附件匹配：`附件.cust_nbr = 047.cust_nbr`（有编码优先编码，客户名兜底须提示重名风险）。
- 月份打横：`SUM(CASE WHEN par_month_id=YYYYMM THEN fee_fm_new ELSE 0 END)`（产数同理用 `fee_cs`）。

### 路径示例（无附件圈定后查客户收入）

```text
Step1 tmp_cust：069/108 等按条件圈 cust_nbr（或从 069 圈 serv_id 再取 cust_nbr）
Step2：047 按 cust_nbr + par_month_id 聚合 sum(fee_fm_new)/sum(fee_cs)
Step3：回连 Step1 或按输出维度打横
```

## 与 048 圈定路径的分工

| 需求 | 圈定表 | 收入表 | 说明 |
|---|---|---|---|
| 圈号后要 **实收** | 069/033 等 | **117** | 本 SC |
| 圈号后要 **客户基本面/产数** | 069/108 等 | **047** | 本 SC |
| 圈号后要 **科目/账目项明细** | 069 | **048** | 不在本 SC；`month_id` 非 `par_month_id` |

同一需求若同时要 **117 实收 + 048 科目**，拆两段 SQL 或 CTAS 多步，勿混表公式。

## 输出字段建议

| 需求 | 117 路径 | 047 路径 |
|---|---|---|
| 号码/接入号 | `acc_nbr` | 通常不直接输出；经 069 带出 |
| 客户编码 | 经 069 补 `cust_nbr` | `cust_nbr` |
| 实收金额 | `ss_amount`（按公式） | — |
| 基本面/产数 | — | `fee_fm_new` / `fee_cs`（已聚合） |
| 月份打横 | `m202501`, `m202502`… | 同上 |

## 风险审计

- 117 与 047/048/069 费用字段混用。
- 047 未先聚合到客户级就回连附件 → 行数膨胀。
- 账期只写开始月、未写闭区间。
- 路径2 圈定条件未写入 Step1，只在 Step2 臆测过滤。
- VPN/专线范围仅来自附件文件名或口头描述，未在 SQL 中显式过滤。
- 048 用 `par_month_id`、117/047 用 `month_id`（字段混用）。

## 自检

- **117**：圈定对象数 vs 117 命中数；`count(*)` vs `count(distinct acc_nbr)`（或 serv_id）；样例 `limit 10` 看 HF/OT 是否合理。
- **047**：聚合前后行数；客户级 `count(distinct cust_nbr)` 与预期一致；打横列非全 NULL。
- **路径1 附件**：输入行数 = 输出行数（LEFT JOIN 保行）或说明 INNER 过滤原因。
- **路径2 CTAS**：Step1 `count(*)` 量级合理；Step2 JOIN 后未异常炸行。

## 错误路径

- 用 047/048/069 查「实收金额」→ 应走 **117**。
- 用 117 查「科目级税后收入」→ 应走 **048**。
- 客户清单产数/基本面用 097 或 069 替代 **047**（除非用户明确 097 专项指标口径）。
- 无附件圈定却写单层 SELECT 把 069 条件硬塞进 117 WHERE（117 上可能没有该维度）。
