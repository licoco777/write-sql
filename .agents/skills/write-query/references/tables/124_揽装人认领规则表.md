# 揽装人认领规则表

- **Hive 表名**: `ads_yz_sales_grid_rule_list`
- **已见案例别名 / 清洗表**: `ads_yz_sales_grid_rule_list_cl`
- **业务定位**: 记录揽装人编码对应的应认领责任田，用于判断号码当前划小责任田是否需要按揽装人认领规则调整。
- **不是**普通揽装人维表、网点维表或揽装所属表：查有效揽装人 / 有效网点仍走 111/112/113。

## 已知字段

| 字段 | 字段含义 | 说明 |
|------|---------|------|
| sales_code | 揽装人编码 / 工号 | 与 069 `sales_code` 关联；可能一人多规则，落 SQL 前须核对粒度 |
| grid_id | 认领规则对应责任田 ID | 目标责任田 |
| grid_code | 认领规则对应责任田编码 | 输出常用字段 |
| grid_name | 认领规则对应责任田名称 | 输出常用字段 |
| grid_type_name | 认领规则对应责任田类型 | 输出常用字段 |
| area_id | 认领规则责任田所属包区 ID | 与 069 当前 `area_id` 做差异判断 |
| subst_id | 认领规则责任田所属分局 ID | 与 069 当前 `subst_id` 做差异判断 |
| subst_name | 认领规则责任田所属分局名称 | 展示字段 |
| branch_id | 认领规则责任田所属营服 ID | 与 069 当前 `branch_id` 做差异判断 |
| branch_name | 认领规则责任田所属营服名称 | 展示字段 |

## 使用规则

- 入口通常是 069 全业务资料表中的 `sales_code`；先按号码范围圈出 `serv_id/acc_nbr/sales_code/当前责任田字段`，再补本表目标责任田。
- 匹配到本表且目标责任田与当前责任田不一致时，才可能进入责任田调整清单。
- `sales_code` 不保证唯一；如果一个揽装人编码命中多条责任田规则，需先按规则版本/有效状态/业务确认去重，不能静默放大号码明细。
- 具体产品范围、机构排除、需求单号码剔除等属于业务参数，不在本表固定。
